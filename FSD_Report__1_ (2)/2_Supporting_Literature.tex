\chapter{SUPPORTING LITERATURE AND RELATED WORK}

\section{Introduction}

This chapter reviews the supporting literature and related technologies that form the foundation of the Vehicle Bidding System project. It covers relevant research, existing systems, and technology frameworks used in building the application.

\section{Web Application Development}

\subsection{Full Stack Development}

Full-stack web development refers to the complete development of both frontend and backend components of a web application. According to industry standards, a full-stack developer is proficient in:

\begin{itemize}
    \item Frontend technologies (HTML, CSS, JavaScript frameworks)
    \item Backend technologies (Server-side languages and frameworks)
    \item Database design and management
    \item Deployment and DevOps practices
\end{itemize}

The MERN stack (MongoDB, Express.js, React.js, Node.js) is one of the most popular full-stack development stacks, allowing developers to use JavaScript across the entire application stack, thus improving productivity and code reusability.

\subsection{Single Page Applications (SPAs)}

Single Page Applications are web applications that load a single HTML page and dynamically update the content as the user interacts with the application. Benefits of SPAs include:

\begin{itemize}
    \item Improved user experience with smooth transitions
    \item Reduced server load through client-side rendering
    \item Faster response times after initial load
    \item Better offline functionality
    \item Easier implementation of progressive web app (PWA) features
\end{itemize}

React.js is a popular JavaScript library for building SPAs, offering component-based architecture and virtual DOM for efficient rendering.

\section{Real-Time Communication Technologies}

\subsection{WebSockets and Socket.io}

WebSockets provide a protocol for two-way communication between client and server over a single TCP connection. Socket.io is a JavaScript library that simplifies WebSocket implementation by:

\begin{itemize}
    \item Providing fallback mechanisms for browsers that don't support WebSockets
    \item Offering automatic reconnection capabilities
    \item Providing room and namespace support for message broadcasting
    \item Enabling real-time event-based communication
\end{itemize}

For the Vehicle Bidding System, Socket.io is essential for delivering real-time bid notifications and enabling live messaging between users.

\section{Database Technologies}

\subsection{NoSQL Databases}

NoSQL databases like MongoDB offer advantages over traditional relational databases:

\begin{itemize}
    \item Flexible schema that adapts to changing requirements
    \item Horizontal scalability through sharding
    \item Better performance for unstructured data
    \item Native support for JSON-like documents
    \item Easier integration with object-oriented programming languages
\end{itemize}

MongoDB specifically provides document-oriented storage, making it ideal for applications with complex, nested data structures like the Vehicle Bidding System.

\subsection{Mongoose ODM}

Mongoose provides an Object Document Mapper (ODM) for MongoDB, offering:

\begin{itemize}
    \item Schema validation at the application level
    \item Built-in methods for common database operations (CRUD)
    \item Support for complex queries and aggregations
    \item Middleware hooks for data processing
    \item Population capabilities for joining related documents
\end{itemize}

\section{Authentication and Security}

\subsection{JWT (JSON Web Tokens)}

JWT is a stateless authentication mechanism that allows secure transmission of user information. Key advantages include:

\begin{itemize}
    \item No server-side session storage required
    \item Scalable for microservices architectures
    \item Works well with cross-domain and CORS requests
    \item Self-contained with all necessary information in the token
    \item Support for cryptographic signature verification
\end{itemize}

\subsection{Password Hashing with Bcrypt}

Bcrypt is an adaptive password hashing algorithm that:

\begin{itemize}
    \item Uses a salt to prevent rainbow table attacks
    \item Has a configurable cost factor for resistance to brute force attacks
    \item Slows down with improved computing power
    \item Is recommended by security professionals
\end{itemize}

\section{API Design and Architecture}

\subsection{RESTful API Principles}

REST (Representational State Transfer) architecture provides guidelines for building scalable, maintainable APIs:

\begin{itemize}
    \item Use HTTP methods (GET, POST, PUT, DELETE) appropriately
    \item Design resources as nouns (not verbs)
    \item Use proper HTTP status codes for responses
    \item Implement proper error handling and validation
    \item Version APIs to support backward compatibility
    \item Document endpoints clearly
\end{itemize}

\subsection{Express.js Framework}

Express.js is a minimal and flexible Node.js web framework that:

\begin{itemize}
    \item Provides easy routing and middleware support
    \item Enables rapid API development
    \item Has a large ecosystem of plugins and extensions
    \item Supports both synchronous and asynchronous operations
    \item Is widely used in production applications
\end{itemize}

\section{Frontend Technologies}

\subsection{React.js}

React.js is a JavaScript library for building user interfaces with several advantages:

\begin{itemize}
    \item Component-based architecture promotes reusability
    \item Virtual DOM improves performance
    \item Unidirectional data flow makes applications predictable
    \item Large community and extensive ecosystem
    \item Excellent developer tools and browser extensions
    \item Can be used for both web and mobile (React Native)
\end{itemize}

\subsection{State Management}

State management in React applications can be handled through:

\begin{itemize}
    \item React Context API for simple state sharing
    \item Redux for complex state management
    \item Zustand or Recoil for lighter weight alternatives
    \item Props drilling for small applications
\end{itemize}
