\chapter{TESTING}

Testing is a crucial phase in software development that ensures the Vehicle Bidding System functions correctly, is reliable, and meets all specified requirements. The testing strategy includes multiple levels of testing to ensure comprehensive coverage and identify defects early in the development cycle.

\section{Introduction to Testing}

Testing involves different types of testing such as unit testing, integration testing, system testing, and user acceptance testing. Each level of testing ensures that different aspects of the system work correctly both individually and as part of the larger system.

\begin{itemize}
    \item \textbf{Unit Testing:} Tests individual components and functions in isolation
    \item \textbf{Integration Testing:} Tests how different components work together
    \item \textbf{System Testing:} Tests the complete system as a whole
    \item \textbf{User Acceptance Testing:} Tests if the system meets user requirements
\end{itemize}

\subsubsection{Security Testing}

\begin{itemize}
    \item \textbf{SQL Injection:} Attempt to inject SQL in search fields - injection blocked
    \item \textbf{XSS Attacks:} Attempt to inject scripts in message fields - scripts sanitized
    \item \textbf{Authentication:} Verify JWT token validation - invalid tokens rejected
    \item \textbf{Authorization:} Non-admin user attempts admin actions - access denied
    \item \textbf{Password Security:} Verify password encryption - passwords never logged in plaintext
    \item \textbf{CORS:} Verify cross-origin requests restricted - unauthorized origins rejected
    \item \textbf{HTTPS:} Verify all communications encrypted - secure protocol enforced
\end{itemize}

\section{Bug Tracking and Resolution}

All discovered bugs are logged in a tracking system with the following information:

\begin{itemize}
    \item Bug ID and date reported
    \item Severity level (Critical, High, Medium, Low)
    \item Description and reproduction steps
    \item Expected vs. actual behavior
    \item Status (Open, In Progress, Fixed, Verified)
    \item Resolution and testing verification
\end{itemize}
